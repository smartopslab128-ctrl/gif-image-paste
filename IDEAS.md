# GIF & Image Paste — 改善案と有料化の候補

## 1. ツールをさらに良くするための機能案

### 使い勝手
- **検索・タグ**: キーワード検索やタグでフィルタ（例: 「仕事」「ネタ」）。有料の「タグ／検索」とセットにしやすい。
- **一括操作**: 複数選択して「まとめてお気に入り」「まとめてタブに追加」「まとめてゴミ箱」。
- **並び替え**: ドラッグ＆ドロップでカードの並びを変更し、その順を保存。
- **プレビュー拡大**: クリックまたはダブルクリックでモーダル／オーバーレイの大きいプレビュー（GIFも再生）。

### 管理・容量
- **フォルダ／コレクション**: カスタムタブの上位概念として「フォルダ」を作成し、その中にタブやアイテムを整理。
- **上限の段階設定**: 無料 50 枚 → 有料で 200 枚、500 枚など。バックアップ／リストア（エクスポートの拡張）。

### 連携・エクスポート
- **クイックエクスポート**: 1クリックで「最後に使ったフォルダ」や「今日の日付フォルダ」にエクスポート。
- **形式変換**: エクスポート時に「GIF→動画」「画像リサイズ」などのオプション（有料にしやすい）。
- **クラウド同期**: 設定で OneDrive / Google Drive 等のフォルダを指定し、ライブラリをそこに保存・読み込み（有料候補）。

### パフォーマンス・品質
- **サムネイルキャッシュ**: 大量枚数でも一覧を軽くするため、サムネイルをディスクにキャッシュ。
- **重複検出の強化**: 類似画像（リサイズや軽い編集後）も「ほぼ同じ」として検出するオプション。

### その他
- **キーボードショートカット**: 一覧で矢印キーで選択、Enter でコピー、Delete でゴミ箱など。
- **テーマ**: ダーク／ライト以外に、アクセント色の変更。
- **多言語**: 英語・日本語の切り替え（公開範囲に応じて）。

## 2. フリーミアムで「有料にすべき」機能の候補

### 有料にしやすいもの（ユーザーが価値を感じやすい）

| 機能 | 理由 |
|------|------|
| **保存枚数上限の引き上げ** | 無料 50 枚、有料 200〜500 枚など。需要が分かりやすく、説明しやすい。 |
| **カスタムタブ数の上限** | 無料は 2〜3 タブまで、有料で無制限。整理したい人に刺さる。 |
| **タグ・キーワード検索** | 枚数が増えると必須になる。「検索できる = 有料」は納得されやすい。 |
| **クラウド同期・バックアップ** | 複数端末や再インストール時の復元。データの安心感で有料化しやすい。 |
| **エクスポートの拡張** | 一括リサイズ、形式変換（GIF→MP4 等）、フォルダ構成の保持など。 |
| **類似画像の重複検出** | ストレージ節約・整理の質向上。上級者向けだが単価を上げやすい。 |

### 無料のままにしやすいもの（獲得・継続のため）

- 基本の追加（クリップボード・ファイル・URL・ドロップ）
- お気に入り、ゴミ箱、基本的なエクスポート（1フォルダにまとめて保存）
- カスタムタブ 1〜2 個程度
- 常に前面表示、トレイ常駐、基本的なソート・サイズ変更

### 料金モデルの例

- **無料**: 50 枚まで、タブ 2 個まで、検索なし、ローカルのみ。
- **有料（月額 or 買い切り）**: 枚数上限アップ、タブ無制限、検索、クラウド同期、高度なエクスポートなど。

「保存枚数」と「タブ数」を有料の軸にすると、実装も比較的簡単で、ユーザーにも伝わりやすいです。

## 3. 広告で収益化する場合（機能的には無償にする場合）

- **メリット**: 全ユーザーに同じ機能を提供できる。課金の心理的ハードルがない。
- **注意点**: Electron デスクトップでは広告の組み込みは Web ほど一般的ではない。表示場所を「設定やサポート画面のみ」に絞ると UX を損ないにくい。
- **折衷案**: 機能は無償のまま「支援／寄付」や「Pro（広告非表示）」のみ有料にする。

現在の実装では保存枚数は **無料 50 / 有料 300**。設定の `isPro: true`（settings.json）で 300 上限にできる。
